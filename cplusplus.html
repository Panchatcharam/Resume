Sequence containers:

  • vector
  • deque
  • list
  • forward_list

vector

  • Stores elements in the contagious memory locations
  • Initially allocates block of memory which is not essentially the same size of number of elements being initialized, can check that using capacity()
  • Performs well if the operation (add/remove) is performed at the end
  • Provides array like indexing
  • Provides amortized time for inserting data basically it is in terms of O(1)
  • If inserting new element causes the vector size to exceed the originally allocated memory, then reallocation will take place to accommodate the new element and the original block will be copied to this new memory location
  • Element can be accessed by offsetting pointer to another element
  • Performs worse if the insert/delete happens at the beginning

    • Complexity

  • Random access - constant O(1)
  • Insertion or removal of elements at the end - amortized constant O(1)
    • Insertion or removal of elements - linear in distance to the end of the vector O(n)

deque

  • Acronym of Double Ended Queue
  • Performs insertion/deletion at both the ends
  • Memory is allocated in chunks but scattered in memory
  • Provides array like indexing elements
  • Returns garbage value if element is accessed by offsetting pointer to another element, since the memory is not allocated in contagious memory locations as in vector
  • If the operation (insert/delete) frequently performed other than beginning/end it performs worse

    • Complexity

  • Random access – O(1)
  • Insert/remove element at the beginning/end – O(1)
  • Insert/remove element – O(n)

list

  • Constant time insert/delete of elements anywhere from the list
  • Implemented as doubly linked list
  • Indexing is not available
  • Performs generally better in inserting, extracting and moving elements in any position within the list for which an iterator has already been obtained

forward_list

  • Implemented as single linked list
  • Indexing is not available

Associative containers:

  • map
  • set
  • multimap
  • multiset

map

  • key value pair, does not allow duplicate keys
  • inserting element using [] will replace/create element
  • inserting using insert() will insert if the key does not exist in map, otherwise fails
  • Underlying data structure is red black tree (binary search tree)
  • Can index by its key to access element (access using [])
  • Slower than the unorder_map which is implemented using hash table

multimap

  • key value pair, does allow duplicate keys
  • does not implement [] operator
  • Underlying data structure is red black tree (binary search tree)

set

  • The value to be stored act as a key
  • The values cannot be modified
  • Elements can be inserted or removed
  • Underlying data structure is red black tree (binary search tree)

Unordered associative containers below are implemented using hash table, thus time complexity id always O(1)

  • unordered_map
  • unordered_set
  • unordered_multimap
  • unordered_multiset
